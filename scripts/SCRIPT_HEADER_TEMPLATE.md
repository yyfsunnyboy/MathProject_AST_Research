# 📋 Scripts 標準標頭模板

## 用途
所有 `scripts/` 目錄下的 Python 腳本都必須遵循此標頭格式，確保：
- 任何人（老師、評審、未來開發者）一看就懂
- 清楚說明研究背景和技術脈絡
- 提供完整的使用說明和範例

---

## 各區塊說明

### 1. 【程式用途】
- **必填**，2-3 句話
- 第一句用最簡單的話說明這個腳本做什麼
- 第二句說明為什麼需要它
- 避免技術術語，用功能導向的語言

### 2. 【研究背景】
- **必填**，建立學術脈絡
- 固定開頭：「專案：旺宏科學獎 - 複合式 AI 架構降低數學題庫生成成本之研究」
- 說明這個腳本在整個研究中的定位
- 連結到核心假設（H1/H2/H3）如果適用

### 3. 【主要功能】
- **必填**，列表形式
- 每個功能包含子項說明
- 優先級由高到低排序
- 可以用數字標號

### 4. 【使用場景】
- **必填**，分三類
- ✅ 必須使用：什麼情況下一定要跑這個腳本
- ⚠️ 建議使用：可選但有幫助的場景
- 🚫 不需要使用：避免誤用

### 5. 【技術說明】
- **必填**，給技術讀者看
- 說明核心技術、演算法、資料流
- 列出重要參數和預設值
- 說明輸入輸出格式

### 6. 【開發流程】
- **選填**，適用於工作流程腳本
- 例如：回歸測試、實驗執行、資料處理
- 提供標準操作程序（SOP）

### 7. 【版本資訊】
- **必填**
- 版本號使用語義化版本（Semantic Versioning）
- 記錄每次重要變更
- 列出相關文件的連結

### 8. 【執行範例】
- **必填**
- 至少提供 2-3 個實際可執行的範例
- 包含預期輸出
- 使用實際的檔案路徑和參數

### 9. 【重要提醒】
- **建議填**
- 預先解答常見問題
- 提供故障排除指引
- 分享最佳實踐

## 標頭撰寫檢查清單

在提交新腳本前，請確認：

- [ ] 檔案開頭有 `# -*- coding: utf-8 -*-`
- [ ] 【程式用途】清楚說明核心功能
- [ ] 【研究背景】連結到旺宏科學獎專案
- [ ] 【主要功能】至少列出 3 項
- [ ] 【使用場景】包含三類情境
- [ ] 【技術說明】涵蓋輸入輸出和依賴
- [ ] 【版本資訊】有版本號和建立日期
- [ ] 【執行範例】可直接複製執行
- [ ] 【重要提醒】預防常見錯誤
- [ ] 整體長度適中（不超過 150 行標頭）

## 何時更新標頭

### 必須更新：
- 新增主要功能
- 修改核心演算法
- 變更輸入輸出格式
- 調整使用流程

### 建議更新：
- 發現常見問題並解決
- 新增使用場景
- 優化參數預設值

### 版本升級規則：
- **Major (X.0)**: 破壞性變更、核心邏輯重寫
- **Minor (X.Y)**: 新增功能、重要改進
- **Patch (X.Y.Z)**: Bug 修復、小調整（可在變更記錄中說明，版本號選填）

## 特殊腳本類型的額外要求

### 1. 實驗執行腳本
額外需要：
- 【實驗設計】區塊：說明 3×3 矩陣或其他實驗配置
- 【資料收集】區塊：說明記錄到資料庫的欄位
- 【統計分析】區塊：說明後續分析方法

### 2. 資料處理腳本
額外需要：
- 【資料格式】區塊：詳細說明輸入輸出的 Schema
- 【資料驗證】區塊：說明資料品質檢查邏輯
- 【錯誤處理】區塊：說明例外情況處理

### 3. 可視化腳本
額外需要：
- 【圖表類型】區塊：說明生成的圖表種類
- 【視覺設計】區塊：說明色彩、字型、尺寸等
- 【匯出格式】區塊：支援的輸出格式（PNG/PDF/HTML）

## 標頭品質標準

### 優秀標頭的特徵：
✅ 非技術人員也能理解用途  
✅ 技術人員能快速掌握實作細節  
✅ 提供可直接複製的執行範例  
✅ 預先回答常見問題  
✅ 連結到相關文件和研究脈絡  

### 需要改進的標頭：
❌ 只有一句話說明  
❌ 充滿技術黑話，無法理解  
❌ 沒有使用範例  
❌ 與專案研究目標脫節  
❌ 版本資訊缺失或過時  

---

**最後提醒**：標頭不只是註解，它是**讓您的研究成果被理解和認可的第一步**。評審老師會從標頭判斷您的研究嚴謹度和專業性！
